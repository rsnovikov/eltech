Описание УТИС экзаменатора


Работа программы в обычном режиме.

При запуске программы появляется окно (рис. 21).

В обычном режиме тестовый файл (проект) выбираются при помощи кнопки . При выборе тестового файла (.TFS) тестируемому предлагается ответить на все вопросы, хранящиеся в нем. При выборе файла проекта (.TFP) – на определенное число вопросов, указанных в нем.



Рис. 21

Выбор вопросов из проекта производится по следующему правилу: из каждого логического файла, входящего в проект, берется одинаковое число вопросов. Например, если в проект включено два файла и требуется выбрать десять вопросов, то из каждого файла будет выбрано по пять вопросов.

Работа программы из командной строки.

Программе с командной строкой можно передать следующие параметры:

1. Имя тестового файла или файла проекта с полным путем до него.
2. Путь в реестре, где программа запишет свой результат (см. 6.5, предоставление результата другим приложениям).

По завершению тестирования выход из программы осуществляется автоматически.

Настройки проэкта по умолчанию.

При загрузке тестового файла (.TFS) настройки проекта принимают значения по умолчанию:

Число вопросов – все вопросы, хранящиеся в тестовом файле;
Время - нет;
Возврат - да;
Метка - да;
Подсчет - разносный;
Стиль отметки - дифференцированный;
Отметка (%) - отлично - 90;
                        хотошо - 75;
                        удовлетворительно - 60;
Отчет - графический, по каждому вопросу.

Переход от одного вопроса к другому.

Переход к предыдущему вопросу осуществляется нажатием кнопки . Если параметр проекта Возврат не установлен, то перейти к предыдущему вопросу нельзя. Переход к следующему вопросу осуществляется нажатием кнопки .

Строка состояния.

Внизу окна расположена строка состояния. Она разделяется на три части:

1. Краткая справка, появляющаяся при удержании курсора мыши над какой-нибудь кнопкой.
2. Номер текущего вопроса / общее число вопросов.
3. – время, оставшееся до окончания тестирования.

Подсчет и вывод результатов тестирования.

Чтобы подсчитать и вывести результаты тестирования нажмите кнопку .

Если установлен параметр проекта Возврат, то перед выводом окончательного результата тестируемому предлагается просмотреть состояние вопросов (рис. 22).



Рис. 22


Пометки напротив вопроса:

– в данном вопросе не был отмечен ни один ответ;
– вопрос был помечен (см. 6.4, создание проекта).

Чтобы перейти к выбранному вопросу нажмите кнопку Перейти. Чтобы вывести результаты тестирования нажмите на кнопку Закончить. В зависимости от параметра проекта Отчет, результат выводится в cтатистическом (рис. 23) или графическом виде (рис. 24).

В верхней части окна указывается тема проекта. В рамке указан результат.

Статистический вид:

Максимальный результат – общее число существенных операций;
Результат, требуемый для сдачи – процент от числа существенных операций, при достижении которого тест считается сданным;
Ваш результат – число существенных операций, набранное тестируемым.



Рис. 23


Графический вид:

Синяя полоска – процент от числа существенных операций, при достижении которого тест считается сданным;
Зеленая полоска – тест сдан;
Красная полоска – тест не сдан.



Рис. 24


При нажатии кнопки Подробно, выводится статистика по каждому вопросу и темам, если установлены соответствующие параметры проекта (рис. 25).



Рис. 25


Предоставление результата другим приложениям.

Вторым параметром в командной строке указывается путь в реестре, куда УТИС экзаменатор должен записать результат тестирования.

Пример:            utise.exe c:\test1.tfp software\tests\test1

Результат тестирования будет записан в HKEY_CURRENT_USER\Software\Tests\Test1\Result

Если параметр проекта Стиль отметки является дифференцированным, то значение параметра Result изменяется от 2 до 5. Если – не дифференцированный, то значение сдал или не сдал.
